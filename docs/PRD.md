# FitGrant — PRD (Product Requirements Document)

> 패션·디자인 분야에 특화된 정부지원사업 매칭 서비스
> v0.1 · 2026-06-13 · MVP 기획

---

## 1. 한 줄 정의

> **"내 브랜드 상황에 맞는 정부지원사업을, 자격요건까지 짚어서 골라주는 패션 특화 매칭 서비스."**

---

## 2. 문제 정의 (Problem)

패션 디자이너·신생 브랜드는 정부지원사업의 **존재 자체를 모르거나, 알아도 자기에게 맞는지 판단하지 못해** 매년 수많은 기회를 놓친다.

- 공고는 KOTRA·KOCCA·중기부·서울산업진흥원 등 **수십 개 기관에 흩어져** 있다.
- 공고문은 행정 용어로 쓰여 있어 **"내가 신청 자격이 되는지"** 판단이 어렵다.
- 비즈인포·기업마당 같은 범용 서비스는 있지만, **패션·디자인 분야 특화 큐레이션과 매칭은 없다.**
- 정작 중요한 **"놓치기 쉬운 자격요건"**(업력 제한, 매출 상한, 중복수혜 불가 등)은 실무 경험자만 안다.

### 왜 지금, 왜 내가 (Founder–Market Fit)

기획자 본인이 **패션 플랫폼에서 정부사업지원 + 해외수출을 직접 담당**해 온 실무자다. "어떤 사업이 어떤 브랜드에 맞는지", "신청서에서 무엇이 당락을 가르는지"를 아는 사람만이 매칭 로직과 데이터 스키마를 제대로 설계할 수 있다. → **도메인 지식 자체가 해자(moat).**

---

## 3. 목표 (Goals)

### 제품 목표
- 브랜드가 **3분 안에** 자기에게 맞는 지원사업 Top 5를 받아본다.
- 각 추천에 **"왜 적합한지 / 놓치기 쉬운 자격요건"**을 함께 제공해 판단을 돕는다.
- 마감 알림으로 **기회를 놓치지 않게** 한다.

### 포트폴리오/커리어 목표
- 도메인 전문성이 드러나는 **데이터 스키마 + 매칭 로직**을 핵심 산출물로 만든다.
- 문제 정의 → 설계 의사결정 → 결과로 이어지는 **케이스 스터디**를 남긴다.

### 창업 확장 가능성 (MVP 범위 밖, 비전)
- B2B 구독(마감 알림·맞춤 리포트), 신청 컨설팅 연계, 수출 트랙 확장.

### 비목표 (Non-Goals, MVP에서 안 하는 것)
- 전 분야 지원사업 망라 (패션·디자인 + 수출에만 집중)
- 공고 전체 자동 크롤링 (MVP는 **수작업 큐레이션** — 오히려 전문성의 증거)
- 신청서 자동 제출/대행
- 회원 간 커뮤니티, 결제

---

## 4. 타깃 사용자 (Persona)

| 페르소나 | 상황 | 핵심 니즈 |
|---|---|---|
| **신진 디자이너 브랜드 대표** (1~3년차) | 자금·인력 부족, 지원사업은 들어봤지만 막막함 | "나도 신청할 수 있는 게 있나?" |
| **소규모 브랜드 운영자** (수출 관심) | 해외 진출하고 싶은데 비용·절차 부담 | "수출 지원·해외전시 사업 뭐가 있지?" |
| **브랜드 사업/기획 담당자** | 여러 공고를 추적·관리해야 함 | "마감 놓치지 않고 관리하고 싶다" |

---

## 5. 핵심 사용자 흐름 (User Flow)

```
랜딩
  → 온보딩 (브랜드 프로필 입력, 5~6문항)
  → 매칭 결과 리스트 (적합도순 Top N + 적합 사유)
  → 사업 상세 카드 (조건 체크리스트, 마감일, 지원규모)
  → 관심 사업 저장 / 마감 알림 설정
```

---

## 6. 기능 명세 (Features)

### P0 — MVP 필수

**F1. 온보딩 프로필 입력**
- 입력 항목: 업력(설립연도), 사업자 유형(개인/법인/예비창업), 연매출 구간, 직원 수, 관심 분야(국내성장/해외수출/전시·박람회/R&D·디자인개발), 수출 경험 유무, 지역
- 결과는 로컬/세션 기반으로도 동작 (로그인 필수 아님)

**F2. 매칭 엔진**
- 프로필 조건 ↔ 사업별 자격요건 매칭 → 적합도 점수 산출
- 룰 기반 매칭(자격 충족/미충족 판정) + Claude API로 **"적합 사유 / 주의할 자격요건"** 자연어 생성
- 결과: 적합도순 정렬, 충족/미충족 조건 시각 표시

**F3. 사업 상세 카드**
- 사업명, 주관기관, 지원 규모/내용, 신청 기간·마감일(D-day), 자격요건 체크리스트, 원문 링크
- 저작권 고려: **요약 + 원문 링크** 제공 (전문 복제 X)

**F4. 마감 알림**
- 관심 사업 저장 → 마감 임박 알림 (MVP: 이메일 또는 PWA 푸시 중 하나만)

### P1 — 여유 시 (버퍼 주차)
- 결과 공유 링크 / 저장된 매칭 다시 보기
- 수출 트랙 강화(국가별 해외전시 일정 연동)
- 간단한 통계("나와 비슷한 브랜드가 많이 신청한 사업")

### P2 — 비전 (MVP 밖)
- B2B 구독 알림, 자동 크롤링, 신청서 작성 보조 AI, 컨설팅 연계

## 7. 데이터 모델 (초안)

> Week 1의 핵심 산출물. 실제 공고 20~30개를 큐레이션하며 스키마를 확정한다.

> **스키마 v0.2** — 서울시 뷰티패션산업과 해외진출 사업 5건을 실제 큐레이션하며 보강함 (`data/programs.seed.json`).

**`programs` (지원사업)**
| 필드 | 타입 | 설명 |
|---|---|---|
| id | text/uuid | |
| title | text | 사업명 |
| agency | text | 주관기관 (예: 서울특별시 뷰티패션산업과) |
| operator | text | 운영기관 (예: 서울패션허브). nullable |
| category | text[] | 해외수출 / 국내성장 / 전시박람회 / 디자인개발 / 자금 / 창업 / R&D / 인력 (v0.2에서 확장) |
| track | text[] | 디자이너 / 기업브랜드 — 한 사업이 트랙별로 자격 다름 ★신규 |
| summary | text | 요약 (직접 작성, 원문 복제 X) |
| support_type | text[] | 현금 / 교육 / 컨설팅 / 공간 / 전시·쇼 / 해외세일즈 등 — 금액만으론 표현 안 되는 지원 형태 ★신규 |
| support_amount | text | 지원 규모/내용 (서술형) |
| support_amount_max | bigint | 현금성 지원 상한(원). 정렬/필터용. nullable ★신규 |
| apply_start / apply_end | date | 신청 기간. nullable |
| apply_cycle | text | 연1회 / 연2회 / 상시 — 마감 후 재모집 알림용 ★신규 |
| source_url | text | 원문 링크 |
| as_of_date | date | 데이터 기준일자 (신뢰성 표시) ★신규 |
| verification | text | confirmed / needs_review — 최신 원문 확인 여부 ★신규 |
| eligibility | jsonb | 자격요건 규칙 (아래) |

**`eligibility` (jsonb 규칙 예시)**
```json
{
  "min_years": 1,             // 업력 하한 ★신규 (예: 아마존 "1년 이상")
  "max_years": 7,             // 업력 상한
  "biz_type": ["법인","개인"],
  "max_revenue": 12000000000, // 매출 상한(원)
  "export_required": false,
  "region": ["서울","전국"], // 소재지 조건
  "notes": "중복수혜 불가, 디자인 분야 우대"
}
```

**`brand_profile` (사용자 입력)** — `eligibility`와 대조해 매칭
- founded_year, biz_type, revenue_range, employees, interests[], has_export, region

**`saved_programs`** — 사용자가 저장/알림 설정한 사업

## 8. 매칭 로직 (개념)

1. **하드 필터**: 자격 미달(업력 초과, 매출 초과, 지역 불일치 등) → "조건 미충족"으로 분리 표시
2. **적합도 점수**: 관심 분야 일치 + 수출 경험 부합 + 마감 임박도 등 가중합
3. **설명 생성(Claude)**: 매칭 결과를 프롬프트로 넘겨 *"왜 적합한지, 신청 시 주의할 자격요건"*을 1~2문장으로 생성 (캐싱으로 비용 절감)

## 9. 기술 스택

- **Frontend/Backend**: Next.js (App Router) — 실제 구현은 최신 안정 버전 16 사용 (2026 기준)
- **Styling/UI**: Tailwind CSS + shadcn/ui
- **DB/Auth**: Supabase (Postgres + Auth + RLS)
- **AI**: Claude API (적합 사유·주의사항 생성) — 응답 캐싱 필수
- **배포**: Vercel
- **PWA**: 마감 알림(푸시) 대응

## 10. 3주 + 1주(버퍼) 실행 플랜

| 주차 | 목표 | 핵심 산출물 | 완료 기준 |
|---|---|---|---|
| **Week 1** | 데이터 모델 + 큐레이션 | 패션·수출 관련 지원사업 20~30개 직접 조사·구조화, `eligibility` 스키마 확정 | Supabase에 사업 데이터 시드 완료, 스키마 문서화 |
| **Week 2** | 매칭 엔진 + 핵심 화면 | 온보딩 폼 → 매칭 결과 → 상세 카드, Claude "적합 사유" 연동 | 프로필 입력 시 적합도순 결과가 사유와 함께 출력 |
| **Week 3** | 알림 + UX 완성도 | PWA 세팅, 마감 알림, 빈상태/로딩/반응형 | 관심 사업 저장→알림, 모바일에서 매끄럽게 동작 |
| **Week 4 (버퍼)** | 배포 + 포트폴리오화 | Vercel 배포, README + 케이스 스터디(문제→설계 의사결정→결과) | 공개 URL + 케이스 스터디 1편 |

## 11. 성공 지표 (포트폴리오 관점)

- ✅ 실제 동작하는 공개 URL (모바일 PWA)
- ✅ 도메인 전문성이 드러나는 데이터 스키마 + 매칭 로직 (코드/문서)
- ✅ 케이스 스터디: "왜 이 문제를, 어떻게 풀었나, 무슨 의사결정을 했나"
- (선택) 지인 브랜드 3~5명 사용 피드백
